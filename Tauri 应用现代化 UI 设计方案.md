### **一、 全局应用壳 (App Shell) 与 Tauri 原生集成**

作为桌面软件，第一印象来自于窗口本身。

#### **1\. 无边框窗口与自定义标题栏 (Frameless Window)**

* **顶部拖拽区：** 隐藏系统默认的丑陋白边标题栏，在 Tauri 中配置 decorations: false。自己用 Vue 实现一个高度约 40px 的顶部栏，赋予 data-tauri-drag-region 属性，使其可以被鼠标拖拽移动。  
* **左侧：** 软件 Logo 与名称。  
* **中间（动态栏）：** 显示当前打开的项目名称或面包屑导航，类似于 VS Code 的顶部栏。  
* **右侧：** 最小化、最大化/还原、关闭的系统控制按钮；旁边放置“全局任务托盘”（显示后台抽帧、转换的进度条环）和“设置”按钮。

#### **2\. 原生系统交互 (Native APIs)**

* **文件选择：** 彻底抛弃浏览器的 \<input type="file"\>，全面使用 Tauri 的 @tauri-apps/api/dialog 唤起系统原生的文件/文件夹选择器，体验更丝滑。  
* **系统通知：** 训练完成或报错时，调用 Tauri 的 Notification API 发送系统级弹窗。  
* **快捷键绑定：** 注册全局快捷键，例如 Ctrl+Space 快速启停推理预览，Ctrl+S 保存当前标注配置。

**二、 响应式布局策略 (中等屏幕 vs. 全屏)**

目标受众可能在 14寸笔记本 (1920x1080，缩放 150%，实际视口约 1280x720) 或 27寸 4K 显示器上工作。软件采用 **流式网格 (Fluid Grid) \+ 弹性面板 (Flex Panels)**。

| 区域 | 中等屏幕 (Windowed, \< 1440px) | 全屏幕 (Fullscreen, \> 1440px) |
| :---- | :---- | :---- |
| **侧边导航栏** | **折叠模式**：仅显示图标，悬浮显示 Tooltip，宽度缩小至 64px，节省横向空间。 | **展开模式**：显示图标 \+ 文字标签，宽度 240px。 |
| **主工作区** | **标签页/堆叠模式**：高级设置、日志面板默认折叠或收纳进 Tabs 中，需要时点击滑出。 | **平铺大屏模式**：所有面板完全展开，日志、图表、配置表单左右/上下分屏并排显示。 |
| **多媒体预览** | **单视图**：图片或视频占据主视角，参数面板以浮窗或右侧窄边栏形式存在。 | **双视窗**：支持“原图”与“检测后结果”左右对比 (Split-view)。 |

**三、 核心页面现代化 UI 细化 (Bento Box 风格)**

Bento Box（便当盒）设计语言强调使用圆角矩形卡片将信息模块化，非常适合重数据的 AI 软件。

#### **页面 1：数据准备 (流式卡片布局)**

不再使用生硬的 Tab，而是采用瀑布流或网格卡片。

* **左侧大卡片 (Dropzone)：** 占据 60% 宽度。大面积的虚线拖拽区，带有悬浮发光效果 (Hover Glow)。拖入视频后，卡片无缝切换为视频播放器与波形/进度条。  
* **右侧控制栈 (Control Stack)：** 上下排列的两到三个圆角卡片。  
  * 卡片 A：抽帧参数配置（现代化的数字步进器、Toggle 开关）。  
  * 卡片 B：清洗过滤配置（双向滑块）。  
* **底部悬浮条 (Action Bar)：** 当用户配置好参数后，底部浮现一个带有毛玻璃效果 (Acrylic/Backdrop-filter) 的操作条，点击“执行任务”。

#### **页面 2：数据集看板 (Dashboard)**

充分利用全屏优势，打造数据指挥中心。

* **顶栏过滤器：** 下拉选择当前加载的数据集版本。  
* **数据总览 (Bento Grid)：**  
  * **左上 (统计)：** 展示图片总数、标注框总数、缺失标注数的统计卡片（大号粗体数字，配合上升/下降趋势图标）。  
  * **右上 (分布图)：** 类别分布的横向条形图，极简风格，隐藏坐标轴，将数值直接写在进度条内。  
  * **下方 (健康度)：** 数据集拆分比例条 (Split Bar)。一个长条，用三种颜色（如蓝、绿、紫）代表 Train/Val/Test 比例，中间带有可拖动的分割线滑块，拖动时实时改变上方 YAML 配置的预览代码块。

#### **页面 3：模型训练台 (沉浸式深色模式)**

这是整个软件最“硬核”的地方，需要类似游戏引擎编辑器的 UI 质感。

* **左侧栏 (配置抽屉)：** 紧凑的表单。预训练模型选择采用“卡片式单选”（每个卡片标明 Nano, Small, Medium 及其对应的预估参数量和速度）。  
* **主视窗 (上下分屏)：**  
  * **上半区 (图表监控)：** 黑底的 ECharts 图表，线条使用荧光色（青色、品红、亮黄），带有辉光效果 (Glow Effect)。展示 Loss 和 mAP。  
  * **下半区 (终端视窗)：** 模拟真实的 Terminal 风格，黑色背景，等宽字体 (如 Fira Code)，不同级别的日志（INFO, WARNING, ERROR）自动赋予不同颜色。  
* **右上角 (硬件监控悬浮窗)：** 类似游戏里的帧率监控条，极简显示 GPU 占用百分比和显存进度条。

#### **页面 4：推理测试 (Theater Mode 剧场模式)**

强调“视觉震撼”和实时反馈。

* **全屏画廊：** 隐藏所有不必要的侧边栏，将内容区域最大化。  
* **顶部悬浮控制条 (Floating HUD)：** 将模型选择、置信度滑块做成一个半透明的药丸状悬浮条，固定在画面顶部中央或底部，不遮挡画面主体。  
* **交互细节：** \* 拖动置信度滑块时，画面中的 BBox（边界框）应该有平滑的出现/消失过渡动画（Fade in/out），而不是生硬的闪烁。  
  * 鼠标悬停在某个 BBox 上时，高亮该框，并弹出一个微型 Tooltip 显示具体的坐标信息和精准置信度。

**四、 前端技术栈与样式库建议 (现代化利器)**

为了快速实现上述现代 UI，建议在 Tauri \+ Vue 3 的基础上，使用以下组合：

1. **CSS 框架：Tailwind CSS**  
   * *理由*：极其适合用来实现复杂的响应式网格和微调间距，暗黑模式 (dark: 修饰符) 的支持开箱即用。  
2. **UI 组件库：Radix Vue 或 Element Plus (定制主题)**  
   * *理由*：如果你想要极客/现代风，推荐使用**无头组件库 (Headless UI)** 如 Radix Vue 配合 Tailwind 自己写样式，或者使用 **Nuxt UI**（虽然是 Nuxt 生态，但在 Vue 3 中也可借鉴其设计）。如果追求速度，使用 Element Plus，但必须全局覆盖其默认的蓝色主题，改为深灰 \+ 强调色。  
3. **图标库：Lucide Icons 或 Phosphor Icons**  
   * *理由*：线条流畅，粗细一致，比老旧的 FontAwesome 现代得多。  
4. **动画库：VueUse (useTransition) 或 Motion One**  
   * *理由*：为 BBox 的出现、面板的展开折叠添加物理级别的丝滑动画。
